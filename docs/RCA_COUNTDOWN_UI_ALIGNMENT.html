Root Cause Analysis

Issue: Countdown Number Misalignment

Date: October 26, 2023
Status: Resolved

1. Issue Description

Problem: During the pre-recording countdown, the countdown numbers (3, 2, 1) were displayed outside the circular progress ring instead of being centered inside it. This created a disjointed and unpolished user interface experience.

Impact:

  • Visual inconsistency in the UI.
  • Reduced readability of the countdown timer.
  • Negative perception of application quality.

2. Root Cause Analysis

2.1. CSS Positioning Conflict Defect

Root Cause: The CSS layout for the .countdown-content container and its children was relying on a mix of relative and absolute positioning that didn't account for the document flow correctly.

  • The .countdown-number was positioned relatively within the flow, pushing it away from the absolutely positioned ring.
  • The transform property in the animation keyframes was overwriting the centering transform needed for absolute positioning.

Code Defect (CSS):

/* Previous CSS causing the issue */
.countdown-number {
    /* Missing absolute positioning to center over the ring */
    margin: 0;
}

@keyframes countdownPulse {
    /* Overwrote any centering transform */
    0% { transform: scale(0.5); opacity: 0; }
    /* ... */
}

3. Solution Implementation

3.1. Absolute Centering Strategy Fix

Fix: We refactored the CSS in src/css/video-fix.css to use absolute positioning for all major elements within the countdown overlay.

Logic:

  1. Container: Made .countdown-content a full-screen flex container to establish a centering context.
  2. Ring & Number: Positioned both .countdown-progress-ring and .countdown-number absolutely in the exact center (top: 50%; left: 50%; transform: translate(-50%, -50%);). This ensures they stack perfectly on top of each other.
  3. Icon & Text: Moved the icon and text to absolute positions at the top (20%) and bottom (25%) respectively, to clear the central area.
  4. Animation: Updated the @keyframes to include the translate(-50%, -50%) property, ensuring the centering is maintained during the pulse animation.

Code Change:

/* New CSS */
.countdown-number {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
    z-index: 2;
}

@keyframes countdownPulse {
    0% {
        transform: translate(-50%, -50%) scale(0.5);
        opacity: 0;
    }
    /* ... */
}
