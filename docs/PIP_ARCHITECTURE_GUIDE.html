Picture-in-Picture (PIP) Architecture Guide

Goal: To explain how the "floating camera" feature works, how we built it, and how the code flows.

1. What is Picture-in-Picture (PIP)?

Picture-in-Picture allows a video to "pop out" of the web page and float on top of other windows. This is crucial for a screen recorder because it lets you see your camera feed even while you are looking at another tab or application you are recording.

Our Approach: We use the browser's native Picture-in-Picture API. This is the most robust method because the OS handles the window, ensuring it stays on top of everything.

2. How We Create the PIP Window

The magic happens in src/js/pipCamera.js. Here is the step-by-step recipe:

Step 1: The Setup (HTML)

We have a hidden video element in our HTML. This element acts as the "source" for the PIP window.

<!-- index.html -->
<div id="pip-camera-overlay">
    <!-- This video element is hidden but holds the camera stream -->
    <video id="pip-camera-video" autoplay playsinline></video>
</div>

Step 2: The Logic (JavaScript)

We use a class called PipCameraManager to handle everything.

The show(cameraStream) Command

This is the main function we call to start PIP.

// src/js/pipCamera.js
async trySystemPip(cameraStream) {
    // 1. Check if the browser supports PIP
    if (!('pictureInPictureEnabled' in document)) {
        return reject('System PIP not supported');
    }

    // 2. Connect the camera stream to our hidden video element
    this.pipVideo.srcObject = cameraStream;

    // 3. IMPORTANT: Mute the video! 
    // Browsers block auto-playing video with sound unless the user clicked it.
    // Since this happens automatically when recording starts, we MUST mute it.
    this.pipVideo.muted = true;

    // 4. Wait for the video to load its metadata (size, duration, etc.)
    this.pipVideo.onloadedmetadata = () => {
        // 5. Play the hidden video
        this.pipVideo.play().then(() => {
            // 6. The Magic Command: Request PIP
            return this.pipVideo.requestPictureInPicture();
        }).then(pipWindow => {
            console.log('Success! Video is now floating.');
        });
    };
}

5. Key Challenges & Solutions

The Autoplay Policy. Modern browsers stop videos from playing automatically if they have sound, to prevent annoying ads. Since our PIP opens automatically when recording starts (without a specific "Open PIP" click), the browser blocks it unless we set muted = true.

Security & OS Control. The System PIP window is managed by your Operating System (Windows/macOS), not the web page. This ensures malicious sites can't fake system dialogs. Therefore, we cannot apply CSS (like border-radius) to the window itself.
