UI Design System & State Flow

Documentation of the visual language and interactive states for the Aarush Screen Recorder application.

1. Color Palette & Semantics

Primary Action

Blue (#0d6efd)
Used for: Main actions, Saving, Sharing

Positive / Ready

Green (#198754)
Used for: Start Recording, Show Video

Danger / Stop

Red (#dc3545)
Used for: Stop Recording, Stop Video

Accent

Purple (#8e44ad)
Used for: Screen Sharing (Distinct action)

Inactive / Disabled

Grey (#6c757d)
Used for: Disabled buttons, Secondary options

2. Button State Flow

The application UI transitions through distinct states based on user interaction. Below is the mapping of button colors and availability for each state.

State 1: Initial Load (Idle)

Application just opened. No media permissions granted yet.

Recording Controls:

State 2: Recording Active

User has clicked "Start Recording". System is capturing video/audio.

Recording Controls:

Note: "Start" becomes disabled (Grey) to prevent double-start. "Stop" becomes active (Red).

State 3: Recording Finished

User clicked "Stop" or screen share ended automatically.

Recording Controls:

Note: "Start" is ready again (Green). "Play" and "Save" become active (Blue) to allow reviewing/saving the file.

3. Special States

Screen Sharing Active

The "Share Screen" button (Purple) is hidden and replaced by "Stop Sharing" (Red).
