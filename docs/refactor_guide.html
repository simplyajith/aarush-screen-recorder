Beginner's Refactoring Guide

A detailed, line-by-line explanation of the architectural changes made to the Aarush Screen Recorder project.

Goal: To help you understand why we changed the code and how the new structure works.

2. Centralized Stream Management New

The Problem: Before, the camera stream and screen stream were like loose change in your pocket—easy to lose track of. Different files (`scripts.js`, `shareScreen.js`) were holding onto them separately.

The Solution: We created a StreamManager. Think of it as a Bank Vault. Everyone gives their streams to the vault, and everyone asks the vault when they need a stream.

Code Deep Dive: src/js/streamManager.js

class StreamManager {
    constructor() {
        // These variables hold our streams. Initially, they are empty (null).
        this.cameraStream = null;
        this.screenStream = null;
        this.activeRecordingStream = null;
    }

    // When the camera turns on, we save it here.
    setCameraStream(stream) {
        this.cameraStream = stream;
    }

    // When screen sharing starts, we save it here.
    setScreenStream(stream) {
        this.screenStream = stream;
    }

    // This is the "Magic Mixer". It combines video and audio for recording.
    createRecordingStream() {
        // Scenario A: User is sharing their screen.
        if (this.screenStream) {
            // We take the video from the screen...
            // AND the audio from the camera/mic (if it exists).
            const tracks = [
                ...this.screenStream.getVideoTracks(),
                ...(this.cameraStream ? this.cameraStream.getAudioTracks() : [])
            ];
            // We create a NEW stream that has both!
            this.activeRecordingStream = new MediaStream(tracks);
        }
        // Scenario B: User is just using the camera (no screen share).
        else if (this.cameraStream) {
            this.activeRecordingStream = this.cameraStream;
        }

        return this.activeRecordingStream;
    }
}

// We create ONE instance of this class for the whole app to use.
const streamManager = new StreamManager();
Why is this better?
Now, screenRecorder.js doesn't need to worry about how to mix the streams. It just calls streamManager.createRecordingStream() and gets the perfect stream ready to record.

3. UI & UX Improvements

3.1. Button State Machine

We made the buttons "smart". They now know when to be clickable (green/red/blue) and when to be disabled (grey).

Example: The "Start Recording" Button

// In screenRecorder.js - When recording STARTS
mediaRecorder.onstart = () => {
    // 1. Disable the Start button so you can't click it twice.
    const startRecordBtn = document.getElementById('start-record');
    startRecordBtn.disabled = true;
    startRecordBtn.classList.add('disabled'); // Makes it look grey

    // 2. Enable the Stop button so you can finish.
    const stopRecordBtn = document.getElementById('stop-record');
    stopRecordBtn.disabled = false;
    stopRecordBtn.classList.remove('disabled'); // Makes it look red (via CSS)
};

4. Code Logic Refinements

4.1. Script Loading Order (The "Undefined" Bug)

The Bug: We saw errors like startRecording is not defined.

The Reason: Imagine trying to read a book before it's been written. The browser was reading scripts.js (which uses the functions) before it read screenRecorder.js (which defines the functions).

The Fix: We moved scripts.js to the bottom of the list in index.html.

<!-- index.html -->
<!-- Load the tools first -->
<script src='src/js/streamManager.js'></script>
<script src='src/js/screenRecorder.js'></script>
...
<!-- Load the main controller LAST -->
<script src='src/js/scripts.js'></script>

4.2. Auto-Stop Logic

We ensured that if you stop sharing your screen, the recording doesn't just freeze—it stops properly and saves your file.

// In shareScreen.js
shareStream.getVideoTracks()[0].addEventListener('ended', () => {
    // If we are currently recording...
    if (mediaRecorder && mediaRecorder.state === 'recording') {
        // ...stop the recording safely.
        stopRecording();
    }
});
