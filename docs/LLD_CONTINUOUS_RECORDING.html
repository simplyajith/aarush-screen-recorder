Low Level Design: Continuous Recording

Design plan to enable seamless transition from Screen+Camera recording to Camera-only recording when screen sharing stops.

Goal: When a user stops screen sharing during a recording session, the recording should NOT stop. Instead, it should seamlessly switch to recording the camera feed, and the PIP overlay should disappear.

1. Current Behavior vs. Desired Behavior

Scenario Current Behavior Desired Behavior
Screen Share Ends Recording stops automatically. Toast notification shown. Recording continues. Video source switches to Camera. PIP hides.
PIP Overlay Visible during screen recording. Disappears immediately when screen share ends.

2. Technical Architecture

2.1. The "Composite Stream" Strategy

We will use a global activeRecordingStream variable. When recording starts with both sources, this stream will contain the screen video track and the mic/camera audio track.

The Switch Mechanism:

When screen sharing ends, we will perform a "hot swap" of tracks within this active stream:

activeRecordingStream.removeTrack(screenVideoTrack);
activeRecordingStream.addTrack(cameraVideoTrack);

3. Implementation Plan

Step 1: Modify screenRecorder.js

1.1 Define activeRecordingStream globally.

1.2 In actuallyStartRecording(), assign the stream being recorded to this variable.

1.3 Implement handleStreamSwitch():

  • Verify mediaRecorder.state === 'recording'.
  • Remove screen track, add camera track.
  • Call pipCamera.hide().
  • Update UI buttons (Show "Share Screen" again).

Step 2: Modify shareScreen.js

2.1 Update handleScreenShareEnded().

2.2 Logic Change:

if (isRecording) {
    handleStreamSwitch(); // Switch to camera, keep recording
} else {
    stopScreenSharing(); // Just cleanup
}

4. Regression Impact Analysis & Mitigation

Risk Area Potential Impact Mitigation / Test Case
High MediaRecorder Stability Recorder might stop or error out when track is removed. TC-001: Verify recording continues for at least 10s after switch.
TC-002: Verify final video file plays back correctly across the switch point.
Medium Audio Sync Audio might drift or cut out during switch. TC-003: Record voice continuously while switching. Check playback for gaps.
Low Button States "Stop Sharing" button might remain visible or "Share Screen" might not reappear. TC-004: Verify "Share Screen" button reappears immediately after switch.
Low PIP Overlay PIP might get stuck on screen. TC-005: Verify PIP disappears instantly.
